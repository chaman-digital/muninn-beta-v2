# MUNINN PROJECT MEMORY BANK 

## 1. Project Context & Objectives
Sistema de Integración de Evidencia Multimodal **«Muninn Beta V1 Operativa»**.
Operando para la consolidación de la **Demanda de Pérdida de Patria Potestad** (Exp. 1784/2022). El motor cruza metadatos de audios transcritos, Capturas de Chat (OCR Local), ubicaciones GPS, y archivos físicos para fundamentar jurídicamente Violencia Vicaria, Violencia Psicoemocional y Alejamiento Justificado por inminente peligro, respaldado bajo los criterios de la SCJN (Autonomía Progresiva).

## 2. Current Status: Snapshot V17.4 (Resiliencia Extrema)

### Resumen de Sesión: Blindaje de Ingesta y Cuotas de Nube (V17.4)

El motor de análisis se enfrentó a los límites físicos y lógicos de la API de Gemini durante el procesamiento de medios y PDFs gigantes (como expedientes de 870 mil caracteres). Se implementaron parches estructurales para asegurar un procesamiento ininterrumpido.

#### 1. Arquitectura Híbrida Evolucionada (Fase 2 + Parches de Resiliencia):
*   **Procesamiento Full Multimodal (Gemini 2.5 Flash):** Imágenes, Audios, Videos y PDFs pesados se procesan nativamente en los servidores de alto rendimiento usando la API `client.files.upload()`.
*   **Parche V17.3 (Control de Cuota Física):** Se detectó y resolvió una fuga de archivos de 20GB. Se implementó una bóveda de seguridad `try...finally` que destruye obligatoriamente todo archivo subido de la nube, ignorando fallas de API. Se ejecutó `limpiar_nube_gemini.py` purgando exitosamente 264 archivos fantasmas bloqueadores.
*   **Parche V17.4 & V17.5 (Resiliencia Extrema):** Desactivado chunking redundante para PDFs gigantes. Implementado mitigador aritmético de "Extra Data" y conversor forzoso de array a dict para fallos nativos de JSON.
*   **Parche V17.6 (Desactivación de Censura):** Anulados los filtros de seguridad (`BLOCK_NONE` en Hate Speech, Harassment, etc) para prevenir la interrupción abrupta de transcripciones y el error 'Unterminated string' en los audios sensibles.
*   **Parche V17.7 (Fallback Nativo):** Alimento redundante para PDFs gigantes. Si la API genera el bloqueo 400 `INVALID_ARGUMENT` por superar sus límites de OCR PDF en "JUICIO FAMILIAR.pdf", se activa un escudo `Try-Except` que intercepta la caída y orquesta una ingesta de "Solo-Texto" usando los 872 mil caracteres extraídos previamente.
*   **Regla Absoluta de Verbatim:** Todos los prompts exigen transcripción literal del 100% del diálogo y los textos para la preservación de la cadena de custodia.
